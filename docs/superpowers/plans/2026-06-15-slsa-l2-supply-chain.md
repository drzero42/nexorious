# SLSA Build L2 + SBOM Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add signed Sigstore provenance (SLSA Build L2) to every released artifact, plus image and source SBOMs, with CI self-verification and a `SECURITY.md`.

**Architecture:** All changes live in `.github/workflows/release-artifacts.yaml` (existing `build`, `image`, `chart` jobs gain attestation steps; one new `verify-attestations` job) plus a new root `SECURITY.md`. New attestation/SBOM/publish steps are gated `if: github.event_name == 'release'` so PR builds are unchanged except for unsigned source-SBOM generation. Provenance uses GitHub's keyless OIDC + Sigstore via `actions/attest-*`.

**Tech Stack:** GitHub Actions, `actions/attest-build-provenance@v4`, `actions/attest-sbom@v4`, `anchore/sbom-action@v0.24.0` (syft), `gh attestation verify`. Validated locally with `actionlint`, `yamllint`, `yq`.

**Spec:** `docs/superpowers/specs/2026-06-15-slsa-l2-supply-chain-design.md`

**Pinned versions** (verified against GitHub releases on 2026-06-15; do not change without re-checking):
- `actions/attest-build-provenance@v4`
- `actions/attest-sbom@v4`
- `anchore/sbom-action@v0.24.0` (pinned to exact patch — pre-1.0, a floating `@v0` would drift across breaking changes)

**House style note:** existing actions are pinned to a major tag (`@v6`, `@v7`); the two `actions/attest-*` follow that. `anchore/sbom-action` is the deliberate exception above.

---

## Why this isn't classic TDD

This is CI configuration with no unit-testable surface. The per-task "test" is:
1. `actionlint` + `yamllint` pass on the workflow, and
2. a `yq` query asserting the new structure exists.

The true end-to-end proof is the first tagged release after merge (the
`verify-attestations` job). Each task still ends in a commit.

### Task 1: SECURITY.md

**Files:**
- Create: `SECURITY.md`

- [ ] **Step 1: Write the file**

Create `SECURITY.md` at the repo root with exactly this content:

```markdown
# Security Policy

## Reporting a vulnerability

Please report security vulnerabilities privately through GitHub's
[private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability):

1. Go to the repository's **Security** tab.
2. Click **Report a vulnerability**.
3. Fill in the advisory form.

Please do **not** open a public issue for security reports.

## Verifying releases

Every released artifact carries signed [SLSA Build L2](https://slsa.dev/spec/v1.0/levels)
provenance generated by GitHub Actions and signed with Sigstore. The container
image and the runtime image's SBOM are attached to the image in `ghcr.io`; the
release binaries, packages, and a source SBOM are attached to the GitHub Release.
Verify with the [`gh` CLI](https://cli.github.com/) (replace `X.Y.Z` with the
release version):

Container image:

    gh attestation verify oci://ghcr.io/drzero42/nexorious:X.Y.Z --repo drzero42/nexorious

A downloaded release binary or package:

    gh attestation verify ./nexorious_X.Y.Z_linux_amd64 --repo drzero42/nexorious

Helm chart:

    gh attestation verify oci://ghcr.io/drzero42/charts/nexorious:X.Y.Z --repo drzero42/nexorious

Inspect the image SBOM:

    gh attestation verify oci://ghcr.io/drzero42/nexorious:X.Y.Z --repo drzero42/nexorious --predicate-type https://spdx.dev/Document

The source SBOM (`nexorious_X.Y.Z_sbom.source.spdx.json`) is attached as a
release asset and is itself covered by the release provenance.
```

Expected: prints `4` (four verify examples).

### Task 2: Provenance + source SBOM in the `build` job

**Files:**
- Modify: `.github/workflows/release-artifacts.yaml` (the `build` job)

- [ ] **Step 1: Grant the build job OIDC + attestation permissions**

The `build` job currently has no `permissions:` block (it inherits the top-level
`contents: read`). Add a job-level block immediately after `runs-on: ubuntu-latest`
under `build:`:

```yaml
  build:
    name: Build binaries and packages
    runs-on: ubuntu-latest
    permissions:
      contents: read
      id-token: write
      attestations: write
    steps:
```

- [ ] **Step 2: Generate the source SBOM into `dist/` before the checksum step**

Insert this step **immediately before** the existing `Checksums over all artifacts`
step (so the SBOM is included in `sha256sums.txt`). It runs on PRs too (unsigned)
to catch syft breakage early:

```yaml
      - name: Generate source SBOM (Go + npm)
        uses: anchore/sbom-action@v0.24.0
        with:
          path: .
          format: spdx-json
          output-file: dist/nexorious_${{ steps.versions.outputs.pkg_version }}_sbom.source.spdx.json
          upload-artifact: false
```

- [ ] **Step 3: Attest provenance for binaries, packages, and the source SBOM**

Insert this step **immediately after** the existing `Checksums over all artifacts`
step (before the upload steps). Release-only:

```yaml
      - name: Attest build provenance (binaries, packages, source SBOM)
        if: github.event_name == 'release'
        uses: actions/attest-build-provenance@v4
        with:
          subject-path: 'dist/nexorious*'
```

### Task 3: Provenance + SBOM in the `image` job

**Files:**
- Modify: `.github/workflows/release-artifacts.yaml` (the `image` job)

- [ ] **Step 1: Add OIDC + attestation permissions to the image job**

The `image` job already has `permissions: { contents: read, packages: write }`.
Extend it to:

```yaml

      contents: read
      packages: write
      id-token: write
      attestations: write
```

- [ ] **Step 2: Give the release build step an `id` so its digest is addressable**

Add `id: build` to the existing `Build and push image (release — multi-arch)`
step (the one gated `if: github.event_name == 'release'`):

```yaml
      - name: Build and push image (release — multi-arch)
        id: build
        if: github.event_name == 'release'
        uses: docker/build-push-action@v7
        with:
          context: .
          build-contexts: |
            binaries=./ci-binaries
          target: runtime-ci
          platforms: linux/amd64,linux/arm64
          push: true
          tags: |
            ghcr.io/drzero42/nexorious:${{ steps.v.outputs.value }}
            ghcr.io/drzero42/nexorious:latest
```

- [ ] **Step 3: Attest image provenance, then generate + attest the image SBOM**

Append these three steps to the **end** of the `image` job's `steps:` list:

```yaml
      - name: Attest image provenance
        if: github.event_name == 'release'
        uses: actions/attest-build-provenance@v4
        with:
          subject-name: ghcr.io/drzero42/nexorious
          subject-digest: ${{ steps.build.outputs.digest }}
          push-to-registry: true

      - name: Generate image SBOM
        if: github.event_name == 'release'
        uses: anchore/sbom-action@v0.24.0
        with:
          image: ghcr.io/drzero42/nexorious@${{ steps.build.outputs.digest }}
          format: spdx-json
          output-file: image-sbom.spdx.json
          upload-artifact: false

      - name: Attest image SBOM
        if: github.event_name == 'release'
        uses: actions/attest-sbom@v4
        with:
          subject-name: ghcr.io/drzero42/nexorious
          subject-digest: ${{ steps.build.outputs.digest }}
          sbom-path: image-sbom.spdx.json
          push-to-registry: true
```

```yaml

        id: push
        run: |
          set -euo pipefail
```

### Task 5: `verify-attestations` job

**Files:**
- Modify: `.github/workflows/release-artifacts.yaml` (append a new job)

- [ ] **Step 1: Append the verify job**

Add this job at the end of the `jobs:` map (sibling of `release-branch`):

```yaml
  verify-attestations:
    name: Verify attestations
    needs: [image, upload, chart]
    if: github.event_name == 'release'
    runs-on: ubuntu-latest
    permissions:
      contents: read
    steps:
      - name: Compute version
        id: v
        run: |
          TAG="${{ github.event.release.tag_name }}"
          echo "value=${TAG#v}" >> "$GITHUB_OUTPUT"

      - name: Log in to ghcr.io
        run: |
          echo "${{ secrets.GITHUB_TOKEN }}" | \
            docker login ghcr.io -u "${{ github.actor }}" --password-stdin

      - name: Download dist artifact
        uses: actions/download-artifact@v8
        with:
          name: dist
          path: dist

      - name: Verify image provenance and SBOM
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          set -euo pipefail
          REF="oci://ghcr.io/drzero42/nexorious:${{ steps.v.outputs.value }}"
          gh attestation verify "$REF" --repo "${{ github.repository }}"
          gh attestation verify "$REF" --repo "${{ github.repository }}" \
            --predicate-type https://spdx.dev/Document

      - name: Verify a release binary
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          set -euo pipefail
          BIN=$(ls dist/nexorious_*_linux_amd64)
          gh attestation verify "$BIN" --repo "${{ github.repository }}"

      - name: Verify chart provenance
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          set -euo pipefail
          gh attestation verify \
            "oci://ghcr.io/drzero42/charts/nexorious:${{ steps.v.outputs.value }}" \
            --repo "${{ github.repository }}"
```
